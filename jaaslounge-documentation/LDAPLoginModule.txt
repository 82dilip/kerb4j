Usage: the module authenticates a given username/password on a Actice Directory Domain Controller via Kerberos/GSSAPI.
If the user exists, the groups of the user are returned as roles. 
You must create a new group, because the module can not retrive in some cases the following group names (e.g. Domain Users, Domain Admins).

The user principal class is org.jaaslounge.UserPrincipal.
The role class is org.jaaslounge.GroupPrincipal.

For general information see JAAS API official documentation.

Web-apps : refer to the documentation of your server. Put jaaslounge-ntlm, jcifs and jcifs-ext jars in the correct classpath.

Example of login module configuration for Tomcat 5.0.x server (jaas.config) :

AD_Realm {
org.jaaslounge.ldaplm.LDAPLoginModule required
mode=tomcat
LDAPServerURL="ldap://192.168.0.1:389"
LDAPSuperUserContext="DC=DOMAIN,DC=AT"
LDAPInitialContextFactory="com.sun.jndi.ldap.LdapCtxFactory"
debug=true;
};

Kerberos {
com.sun.security.auth.module.Krb5LoginModule required client=false useTicketCache=false useSubjectCredsOnly=true useKeyTab=false;
};

security descriptor (<myapp>.xml or server.xml) :

<Realm className="org.apache.catalina.realm.JAASRealm"
appName="AD_Realm"
roleClassNames="org.jaaslounge.GroupPrincipal"
userClassNames="org.jaaslounge.UserPrincipal"
debug="0"
useContextClassLoader="false"/>


Kerberos configuration (krb.ini) :

[libdefaults]
        default_realm = DOMAIN.AT
        default_tgs_enctypes = des-cbc-md5
        default_tkt_enctypes = des-cbc-md5

[realms]
DOMAIN.AT = {
        kdc = HOSTNAME.DOMAIN.AT:88
        kpasswd = HOSTNAME.DOMAIN.AT
}


[domain_realm]
        .domain.at = DOMAIN.AT


Known bugs and limitations :
- You must create a new group, because the module can not retrieve in some cases the following group names (e.g. Domain Users, Domain Admins).
